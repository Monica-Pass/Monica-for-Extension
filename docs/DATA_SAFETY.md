# Chrome/Edge 商店数据安全申报底稿

此文件是提交商店问卷时的事实底稿。最终选项应与待提交版本的 Manifest 和代码再次核对。

## 单一用途

安全保存、同步并在用户明确确认后填充登录凭据、TOTP、身份、地址、支付信息和 Passkey。

## 数据类别

| 商店类别 | 是否处理 | 处理方式与目的 |
| --- | --- | --- |
| 身份验证信息 | 是 | 用户名、密码、TOTP、Passkey；用于保存、同步、登录和自动填充。 |
| 个人身份信息 | 是（用户可选） | 姓名、邮箱、证件、地址；用于用户创建的身份项目和表单填充。 |
| 财务与支付信息 | 是（用户可选） | 银行卡、银行/支付账号；用于用户明确点击后的表单填充和同步。 |
| 网站内容 | 是，瞬时 | 读取当前页面 URL/标题和表单结构；捕获用户刚提交的登录候选以显示保存提示。 |
| 浏览历史 | 否 | 只处理当前活动页面/frame，不建立浏览历史数据集。 |
| 精确/粗略位置、健康、通讯、用户活动分析 | 否 | 功能不读取或推断这些类别。用户自行保存在备注中的自由文本仍属于密码库内容。 |
| 诊断信息 | 是，本地 | 保存脱敏的 Provider 操作结果；不自动上传，仅用户主动导出。 |

## 传输与共享申报

- 默认本地模式：密码库只以 AES-256-GCM 信封保存在本机 IndexedDB，不传给 Monica-Pass。
- WebDAV（可选）：向用户填写的服务器发送 Basic Auth，并下载/上传 Android 兼容密码库快照。无备份密码时快照为普通 ZIP；有备份密码时为 `MONICA_ENC_V1` 加密内容。
- Bitwarden（可选）：向用户选择的官方或自托管服务发送认证请求和加密 Cipher 同步/CRUD 请求。
- 自动填充：用户点击后把选中项目的必要字段写入目标网页；目标网站因此能读取其表单值。
- 不出售数据，不向广告商或数据经纪商共享，不用于广告、信用评估或与单一用途无关的分析。

为审慎申报，凡数据会离开本机前往用户选择的 WebDAV、Bitwarden 或目标网站，均按“会传输/共享给用户请求的服务”披露，不以“用户自己配置服务器”为由回答成“完全本地处理”。

## 安全实践

- 本地密码库：PBKDF2-HMAC-SHA256（600,000 次）和 AES-256-GCM。
- 解锁会话：`chrome.storage.session`，仅可信扩展上下文可访问，并自动过期。
- Provider 凭据、Token、Vault Key、缓存项目和可导出 Passkey 私钥：只持久化于本地加密信封。
- WebDAV：远程地址强制 HTTPS；只有精确的 localhost、127.0.0.0/8 或 `[::1]` 回环开发地址允许 HTTP。凭据请求禁止重定向。
- Bitwarden：远程地址强制 HTTPS，仅 localhost/127.0.0.1 允许 HTTP。
- 不执行远程代码；运行时代码、WASM、字体和资源全部包含在发布包内。
- 不自动上传日志或崩溃报告；脱敏诊断由用户主动下载并自行决定是否分享。

## 商店问卷建议答案

- 数据用途：应用功能、安全与防欺诈（来源/RP ID/frame 核验）；不选择广告或个性化营销。
- 数据出售：否。
- 与单一用途无关的数据使用：否。
- 人工查看数据：项目维护者无法访问本地库；只有用户主动把导出文件提供给他人时，对方才可能查看。
- 远程代码：否。
- 隐私政策：使用仓库内 [PRIVACY.md](PRIVACY.md) 的公开页面 URL。
